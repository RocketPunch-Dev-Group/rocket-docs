로켓펀치 Open API 체인지로그는 모든 릴리스, 신규 엔드포인트, 동작 변경, 지원 중단 안내를 한곳에 기록한다. 이 페이지를 북마크하고 연동을 업데이트할 때마다 확인한다. 항목은 최신순으로 정렬된다.

<Note>
  로켓펀치 Open API는 현재 **초대 빌더 베타(Invited Builder Beta)** 단계다. API 표면은 승인된 빌더에게 안정적이고 프로덕션에 사용할 수 있지만, 베타가 진행되면서 신규 엔드포인트와 기능이 추가된다. 변경 사항을 알림받으려면 [developers.rocketpunch.com](https://developers.rocketpunch.com)에서 공지를 구독한다.
</Note>

## v1 — 빌더 베타

**릴리스: 2025**

초대 빌더에게 제공되는 로켓펀치 Open API의 최초 공개 릴리스. 이 버전은 향후 버전이 기반으로 삼을 기초 엔드포인트 집합, 인증 모델, 데이터 스키마를 확립한다.

### 엔드포인트

### 인증

- **App Key 인증** — `X-RP-API-Key` 요청 헤더를 통한 방식. 키는 `rp_app_xxxxxxxxxxxxxxxx` 형식을 따르며 개발자 포털에서 발급된다.
- **OAuth 2.0 Authorization Code Flow** — 사용자 동의 데이터 접근용. [developers.rocketpunch.com/apps/new](https://developers.rocketpunch.com/apps/new)에서 리디렉션 URI를 등록하고 [Auth Flow 가이드](/api/oauth/flow)에 설명된 4단계 흐름을 구현한다.

### 플랫폼 기능

- **다국어 지원** — 모든 엔드포인트가 `lang` 파라미터를 받는다. 모든 로케일에서 일관된 응답 스키마로 11개 언어를 지원한다: 한국어, 영어, 일본어, 중국어(간체), 중국어(번체), 스페인어, 프랑스어, 독일어, 포르투갈어, 인도네시아어, 베트남어.
- **99.95% 가용성 SLA** — API는 초대 빌더 파트너를 위해 99.95% 가동률을 보장한다.
- **페이지네이션된 응답** — 모든 목록 엔드포인트는 응답 엔벨로프에 `page`, `per_page`, `total_pages` 필드가 포함된 페이지네이션 결과를 반환한다.
- **레이트 리밋 헤더** — 모든 응답에 `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` 헤더가 포함된다. [레이트 리밋](/api/rate-limits) 페이지를 참고한다.

## 버저닝 정책

로켓펀치 API는 **URL 기반 버저닝**을 사용한다. 현재 버전은 `v1`이며 모든 엔드포인트 경로에 반영된다:

```text
https://openapi.rocketpunch.com/api/v1/{resource}
```

### 무엇이 새 버전을 유발하는가

기존 연동을 깨뜨리는 변경이 있을 때 새 메이저 버전(예: `v2`)이 릴리스된다. 파괴적 변경의 예:

- 엔드포인트 제거
- 필수 요청 파라미터의 이름 변경 또는 제거
- 응답 필드의 데이터 타입 변경
- 응답 스키마에서 필드 제거

### 무엇이 새 버전을 유발하지 않는가

추가적 변경은 버전 상향 없이 현재 버전 내에서 이뤄진다. 이런 변경은 하위 호환되며 기존 연동을 깨뜨리지 않는다:

- 새 엔드포인트 추가
- 선택적 요청 파라미터 추가
- 응답 객체에 새 필드 추가
- 기존 파라미터에 새 지원 값 추가(예: 새 `lang` 코드)
- 성능 개선 및 버그 수정

### 지원 중단 정책

버전이 폐기 예정일 때 로켓펀치는:

1. 개발자 포털과 등록된 앱 소유자 이메일을 통해 최소 **6개월** 전에 지원 중단 날짜를 공지한다.
2. 전체 공지 기간 동안 지원 중단된 버전을 계속 제공한다.
3. 폐기일이 다가오면 지원 중단된 버전의 모든 응답에 `Deprecation` 헤더를 반환한다.

<Tip>
  신규 엔드포인트, 베타 기능, 예정된 릴리스에 대한 공지는 [developers.rocketpunch.com](https://developers.rocketpunch.com)에서 확인한다. 개발자 포털에 링크된 RSS 피드로 체인지로그를 팔로우할 수도 있다.
</Tip>
