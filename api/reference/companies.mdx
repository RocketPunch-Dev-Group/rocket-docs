Companies 엔드포인트는 17만\+개 기업 프로필로 이루어진 로켓펀치 색인에 대한 검색 접근을 제공한다. 회사명, 설명 키워드, 산업 용어로 쿼리해 창업 정보, 임직원 수, 웹사이트 URL을 포함한 구조화된 프로필 데이터를 조회할 수 있다. 이 엔드포인트는 기업 디렉터리 구동, CRM 레코드 보강, 투자자 리서치 도구 구축, 또는 채용 공고와 함께 고용주 맥락을 노출하는 데 적합하다.

## 엔드포인트

```text
GET https://openapi.rocketpunch.com/api/v1/companies
```

## 요청 헤더

| 헤더 | 필수 | 설명 |
| --- | --- | --- |
| `X-RP-API-Key` | 예 | `rp_app_xxxxxxxxxxxxxxxx` 형식의 App Key. [개발자 포털](https://developers.rocketpunch.com)에서 발급받는다. |

## 쿼리 파라미터

<ParamField query="keyword" type="string">
  기업을 필터링하는 검색어. 회사명과 설명에 매칭된다. 예: `fintech`, `kakao`, `SaaS`.
</ParamField>

<ParamField query="lang" default="ko" type="string">
  응답 콘텐츠의 언어 코드. 지원되는 값으로는 `ko`(한국어), `en`(영어)이 있으며 총 11개 언어를 지원한다. 응답 스키마는 모든 언어에서 일관된다.
</ParamField>

<ParamField query="page" default="1" type="integer">
  조회할 페이지 번호. 페이지 번호는 `1`부터 시작한다. 단일 페이지보다 큰 결과 집합을 페이지네이션하려면 `per_page`와 함께 사용한다.
</ParamField>

<ParamField query="per_page" type="integer">
  페이지당 반환할 기업 프로필 수. 페이로드 크기와 연동에 필요한 API 호출 수 사이의 균형을 맞추도록 조정한다.
</ParamField>

## 요청 예시

<CodeGroup>

```javascript JavaScript
const response = await fetch(
  "https://openapi.rocketpunch.com/api/v1/companies?keyword=fintech&lang=en",
  {
    method: "GET",
    headers: {
      "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
    },
  }
);

const data = await response.json();
console.log(data.items);
```

```python Python
import requests

url = "https://openapi.rocketpunch.com/api/v1/companies"
params = {
    "keyword": "fintech",
    "lang": "en",
}
headers = {
    "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
}

response = requests.get(url, params=params, headers=headers)
data = response.json()
print(data["items"])
```

```bash cURL
curl -G "https://openapi.rocketpunch.com/api/v1/companies" \
  --data-urlencode "keyword=fintech" \
  --data-urlencode "lang=en" \
  -H "X-RP-API-Key: rp_app_xxxxxxxxxxxxxxxx"
```

</CodeGroup>

## 응답

성공한 요청은 HTTP `200`과 함께 `items` 키 아래에 페이지네이션된 기업 프로필 목록이 담긴 JSON 본문을 반환한다.

```json
{
  "items": [
    {
      "id": "co_04kp2m",
      "name": "Toss",
      "handle": "toss",
      "description": "Korea's leading fintech super-app providing payments, banking, and investment services.",
      "employee_count": 1200,
      "founded_year": 2013,
      "website": "https://toss.im"
    }
  ],
  "total": 872,
  "page": 1,
  "per_page": 20
}
```

<Note>
  위에 표시된 필드 값은 예시일 뿐이다. 실제 값은 쿼리 시점의 실시간 데이터를 반영한다. `employee_count`, `founded_year` 같은 필드는 기업이 로켓펀치 프로필에 해당 정보를 제공하지 않은 경우 `null`일 수 있다.
</Note>

### 응답 필드

<ResponseField name="items" type="array" required>
  쿼리에 일치하는 기업 프로필 객체 배열. 주어진 `keyword`에 일치하는 기업이 없으면 빈 배열을 반환한다.

  <Expandable title="기업 객체 필드">
    <ResponseField name="items[].id" type="string">
      로켓펀치에서 기업의 고유 식별자.
    </ResponseField>

    <ResponseField name="items[].name" type="string">
      로켓펀치 프로필에 표시되는 기업의 표시 이름.
    </ResponseField>

    <ResponseField name="items[].handle" type="string">
      기업의 로켓펀치 프로필에 대한 URL-safe 핸들. `https://www.rocketpunch.com/companies/{handle}`로 프로필 직접 링크를 구성할 수 있다.
    </ResponseField>

    <ResponseField name="items[].description" type="string">
      기업이 제공한 사업 또는 미션에 대한 짧은 설명.
    </ResponseField>

    <ResponseField name="items[].employee_count" type="integer">
      대략적인 임직원 수. 자가 보고 값이며 제공되지 않은 경우 `null`일 수 있다.
    </ResponseField>

    <ResponseField name="items[].founded_year" type="integer">
      기업이 설립된 네 자리 연도. 제공되지 않은 경우 `null`일 수 있다.
    </ResponseField>

    <ResponseField name="items[].website" type="string">
      기업의 공식 웹사이트 URL. 제공되지 않은 경우 `null`일 수 있다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer" required>
  모든 페이지에 걸쳐 쿼리에 일치하는 기업 프로필 총 개수.
</ResponseField>

<ResponseField name="page" type="integer" required>
  이 응답에 반영된 현재 페이지 번호.
</ResponseField>

<ResponseField name="per_page" type="integer" required>
  이 응답에 반영된 페이지당 결과 수.
</ResponseField>

## 오류 응답

| HTTP 상태 | 의미 |
| --- | --- |
| `401 Unauthorized` | `X-RP-API-Key`가 없거나 유효하지 않다. |
| `429 Too Many Requests` | 플랜의 요청 한도를 초과했다. |
| `500 Internal Server Error` | 예상치 못한 서버 측 오류가 발생했다. 지수 백오프로 재시도한다. |

<Tip>
  Companies와 Jobs 엔드포인트를 결합해 채용 공고에 전체 기업 맥락을 보강한다. `/api/v1/jobs`에서 채용 공고를 조회한 뒤, 기업의 `handle`을 `/api/v1/companies?keyword={handle}`로 조회해 사용자에게 더 풍부한 고용주 정보를 표시한다.
</Tip>
