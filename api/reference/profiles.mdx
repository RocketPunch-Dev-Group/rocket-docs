Profiles 엔드포인트는 전문 헤드라인, 스킬, 현재 소속 기업을 포함한 로켓펀치 사용자 프로필 데이터에 대한 접근을 제공한다. 공개 프로필 데이터는 App Key만으로 접근할 수 있어, 후보자 검색, 스킬 매칭, 디렉터리 기능을 손쉽게 만들 수 있다. 연락처나 비공개 경력 같은 비공개 또는 사용자별 프로필 정보에 접근하려면, 요청에 사용자의 명시적 동의로 획득한 OAuth 액세스 토큰이 있어야 한다.

## 엔드포인트

```text
GET https://openapi.rocketpunch.com/api/v1/profiles
```

## 인증

<Note>
  이 엔드포인트는 필요한 데이터에 따라 두 가지 인증 모드를 지원한다:

  - **App Key만** — 공개적으로 이용 가능한 프로필 데이터를 반환한다. 검색, 발견, 디렉터리 활용 사례에 적합하다.
  - **OAuth 액세스 토큰** — 비공개 경력이나 연락처 등 사용자가 승인한 추가 프로필 필드를 반환한다.

  OAuth를 설정하고 사용자 토큰을 획득하려면 [OAuth 개요](/api/oauth/overview)를 참고한다.
</Note>

## 요청 헤더

App Key로 인증할 때는 다음을 포함한다:

| 헤더 | 필수 | 설명 |
| --- | --- | --- |
| `X-RP-API-Key` | 예 (App Key 모드) | `rp_app_xxxxxxxxxxxxxxxx` 형식의 App Key. |

사용자를 대신해 OAuth 토큰으로 인증할 때는 App Key 헤더를 다음으로 대체한다:

| 헤더 | 필수 | 설명 |
| --- | --- | --- |
| `Authorization` | 예 (OAuth 모드) | `Bearer {oauth_token}` 형식의 Bearer 토큰. |

## 쿼리 파라미터

<ParamField query="keyword" type="string">
  프로필을 필터링하는 검색어. 사용자 이름, 전문 헤드라인, 나열된 스킬에 매칭된다. 예: `React`, `data scientist`, `product designer`.
</ParamField>

<ParamField query="lang" default="ko" type="string">
  응답 콘텐츠의 언어 코드. 일반적인 값은 `ko`(한국어), `en`(영어)이다. 로켓펀치는 11개 언어를 지원하며 언어와 관계없이 일관된 스키마를 반환한다.
</ParamField>

<ParamField query="page" default="1" type="integer">
  조회할 페이지 번호. 페이지 번호는 `1`부터 시작한다.
</ParamField>

<ParamField query="per_page" type="integer">
  페이지당 반환할 프로필 수. 연동의 처리량과 페이로드 요구에 맞게 조정한다.
</ParamField>

## 요청 예시

<CodeGroup>

```javascript JavaScript
// App Key 인증 — 공개 프로필
const response = await fetch(
  "https://openapi.rocketpunch.com/api/v1/profiles?keyword=React&lang=en",
  {
    method: "GET",
    headers: {
      "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
    },
  }
);

const data = await response.json();
console.log(data.items);
```

```python Python
import requests

url = "https://openapi.rocketpunch.com/api/v1/profiles"
params = {
    "keyword": "React",
    "lang": "en",
}
headers = {
    # 공개 데이터에는 X-RP-API-Key를, OAuth에는 Authorization을 사용
    "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
}

response = requests.get(url, params=params, headers=headers)
data = response.json()
print(data["items"])
```

```bash cURL
# App Key 인증 — 공개 프로필
curl -G "https://openapi.rocketpunch.com/api/v1/profiles" \
  --data-urlencode "keyword=React" \
  --data-urlencode "lang=en" \
  -H "X-RP-API-Key: rp_app_xxxxxxxxxxxxxxxx"

# OAuth 인증 — 사용자가 승인한 프로필 데이터
curl -G "https://openapi.rocketpunch.com/api/v1/profiles" \
  --data-urlencode "keyword=React" \
  --data-urlencode "lang=en" \
  -H "Authorization: Bearer {oauth_token}"
```

</CodeGroup>

## 응답

성공한 요청은 HTTP `200`과 함께 `items` 키 아래에 페이지네이션된 사용자 프로필 객체 목록이 담긴 JSON 본문을 반환한다.

```json
{
  "items": [
    {
      "handle": "jane-doe",
      "name": "Jane Doe",
      "headline": "Frontend Engineer · React · TypeScript",
      "skills": ["React", "TypeScript", "Next.js", "GraphQL"],
      "current_company": {
        "name": "Rocketpunch Labs",
        "handle": "rocketpunch-labs"
      }
    }
  ],
  "total": 512,
  "page": 1,
  "per_page": 20
}
```

<Note>
  위에 표시된 필드 값은 예시일 뿐이다. 프로필 데이터는 각 사용자의 개인정보 설정에 따른다. 사용자가 해당 정보를 공개하지 않았거나 비공개 필드에 대한 OAuth 동의가 부여되지 않은 프로필의 경우, 필드는 `null`이거나 없을 수 있다.
</Note>

### 응답 필드

<ResponseField name="items" type="array" required>
  쿼리에 일치하는 사용자 프로필 객체 배열. 일치하는 프로필이 없으면 빈 배열을 반환한다.

  <Expandable title="프로필 객체 필드">
    <ResponseField name="items[].handle" type="string">
      사용자의 로켓펀치 프로필에 대한 URL-safe 핸들. `https://www.rocketpunch.com/@{handle}`로 직접 링크를 구성할 수 있다.
    </ResponseField>

    <ResponseField name="items[].name" type="string">
      사용자의 표시 이름. 사용자가 개인정보 설정에서 이름 공개를 제한한 경우 `null`일 수 있다.
    </ResponseField>

    <ResponseField name="items[].headline" type="string">
      사용자가 제공한 짧은 전문 헤드라인. 보통 직무, 전문성, 핵심 스킬을 설명한다.
    </ResponseField>

    <ResponseField name="items[].skills" type="array of strings">
      사용자가 프로필에 추가한 스킬 목록. 스킬을 나열하지 않았거나 스킬이 공개되지 않은 경우 빈 배열을 반환한다.
    </ResponseField>

    <ResponseField name="items[].current_company" type="object">
      프로필에 나열된 사용자의 현재 고용주 요약. 현재 재직 정보를 제공하지 않은 경우 `null`을 반환한다.

      <Expandable title="현재 기업 필드">
        <ResponseField name="items[].current_company.name" type="string">
          사용자의 현재 고용주 표시 이름.
        </ResponseField>

        <ResponseField name="items[].current_company.handle" type="string">
          현재 고용주의 로켓펀치 핸들. 추가 맥락을 위해 [Companies API](/api/reference/companies)와 함께 사용할 수 있다.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer" required>
  모든 페이지에 걸쳐 쿼리에 일치하는 프로필 총 개수.
</ResponseField>

<ResponseField name="page" type="integer" required>
  이 응답에 반영된 현재 페이지 번호.
</ResponseField>

<ResponseField name="per_page" type="integer" required>
  이 응답에 반영된 페이지당 결과 수.
</ResponseField>

## 오류 응답

| HTTP 상태 | 의미 |
| --- | --- |
| `401 Unauthorized` | `X-RP-API-Key` 또는 OAuth 토큰이 없거나 유효하지 않다. |
| `403 Forbidden` | 유효한 OAuth 토큰 없이 비공개 프로필 데이터에 접근하려 했거나, 사용자가 동의하지 않았다. |
| `429 Too Many Requests` | 플랜의 요청 한도를 초과했다. |
| `500 Internal Server Error` | 예상치 못한 서버 측 오류가 발생했다. 지수 백오프로 재시도한다. |

<Warning>
  프로필 데이터는 개인정보다. 이 엔드포인트가 반환하는 모든 데이터는 관련 개인정보 보호법과 로켓펀치 API 이용약관에 따라 처리해야 한다. 애플리케이션의 명시된 목적에 필요한 범위를 넘어 사용자 프로필 데이터를 캐시하거나 저장하지 않는다.
</Warning>
