Events 엔드포인트는 로켓펀치에 등록된 IT 네트워킹 이벤트, 컨퍼런스, 강연, 초대 전용 밋업을 조회하게 해준다. 키워드로 검색하고 언어로 필터링해 사용자에게 관련 이벤트를 노출한 뒤, 그 데이터를 캘린더, 알림 서비스, 커뮤니티 플랫폼, 이벤트 애그리게이터에 임베드할 수 있다. 결과는 페이지네이션되며 다른 로켓펀치 API 엔드포인트와 동일한 일관된 구조를 따른다.

## 엔드포인트

```text
GET https://openapi.rocketpunch.com/api/v1/events
```

## 요청 헤더

| 헤더 | 필수 | 설명 |
| --- | --- | --- |
| `X-RP-API-Key` | 예 | `rp_app_xxxxxxxxxxxxxxxx` 형식의 App Key. [개발자 포털](https://developers.rocketpunch.com)에서 발급받는다. |

## 쿼리 파라미터

<ParamField query="keyword" type="string">
  이벤트를 필터링하는 검색어. 이벤트 제목, 설명, 주최자 이름에 매칭된다. 예: `AI`, `startup`, `design`.
</ParamField>

<ParamField query="lang" default="ko" type="string">
  응답 콘텐츠의 언어 코드. 일반적인 값은 `ko`(한국어), `en`(영어)이다. 로켓펀치는 11개 언어를 지원하며 모든 언어에서 일관된 스키마를 반환한다.
</ParamField>

<ParamField query="page" default="1" type="integer">
  조회할 페이지 번호. 페이지 번호는 `1`부터 시작한다. 큰 결과 집합을 페이지네이션하려면 `per_page`와 함께 사용한다.
</ParamField>

<ParamField query="per_page" type="integer">
  페이지당 반환할 이벤트 수. 페이로드 크기와 전체 데이터 순회에 필요한 요청 수를 제어하도록 조정한다.
</ParamField>

## 요청 예시

<CodeGroup>

```javascript JavaScript
const response = await fetch(
  "https://openapi.rocketpunch.com/api/v1/events?keyword=AI&lang=en",
  {
    method: "GET",
    headers: {
      "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
    },
  }
);

const data = await response.json();
console.log(data.items);
```

```python Python
import requests

url = "https://openapi.rocketpunch.com/api/v1/events"
params = {
    "keyword": "AI",
    "lang": "en",
}
headers = {
    "X-RP-API-Key": "rp_app_xxxxxxxxxxxxxxxx",
}

response = requests.get(url, params=params, headers=headers)
data = response.json()
print(data["items"])
```

```bash cURL
curl -G "https://openapi.rocketpunch.com/api/v1/events" \
  --data-urlencode "keyword=AI" \
  --data-urlencode "lang=en" \
  -H "X-RP-API-Key: rp_app_xxxxxxxxxxxxxxxx"
```

</CodeGroup>

## 응답

성공한 요청은 HTTP `200`과 함께 `items` 키 아래에 페이지네이션된 이벤트 객체 목록이 담긴 JSON 본문을 반환한다.

```json
{
  "items": [
    {
      "id": "evt_07rq4n",
      "title": "Korea AI Summit 2025",
      "event_type": "conference",
      "start_date": "2025-03-15T09:00:00Z",
      "end_date": "2025-03-16T18:00:00Z",
      "location": "COEX, Seoul, South Korea",
      "organizer": {
        "name": "Korea AI Association",
        "handle": "korea-ai-association"
      }
    }
  ],
  "total": 148,
  "page": 1,
  "per_page": 20
}
```

<Note>
  위에 표시된 필드 값은 예시일 뿐이다. 실제 값은 쿼리 시점의 실시간 이벤트 데이터에 따라 달라진다. 주최자가 해당 정보를 제공하지 않은 이벤트의 경우 `end_date`, `location` 같은 필드는 `null`일 수 있다.
</Note>

### 응답 필드

<ResponseField name="items" type="array" required>
  쿼리에 일치하는 이벤트 객체 배열. 주어진 키워드에 일치하는 이벤트가 없으면 빈 배열을 반환한다.

  <Expandable title="이벤트 객체 필드">
    <ResponseField name="items[].id" type="string">
      로켓펀치에서 이벤트의 고유 식별자.
    </ResponseField>

    <ResponseField name="items[].title" type="string">
      주최자가 등록한 이벤트의 전체 표시 제목.
    </ResponseField>

    <ResponseField name="items[].event_type" type="string">
      이벤트 성격을 설명하는 카테고리. 일반적인 값은 다음과 같다:

      - `conference` — 대규모 멀티 세션 이벤트
      - `networking` — 자유롭거나 구조화된 네트워킹 밋업
      - `lecture` — 교육 강연 또는 세미나
      - `exclusive` — 초대 전용 또는 제한된 접근 이벤트
    </ResponseField>

    <ResponseField name="items[].start_date" type="string">
      이벤트 시작 시점의 ISO 8601 타임스탬프.
    </ResponseField>

    <ResponseField name="items[].end_date" type="string">
      이벤트 종료 시점의 ISO 8601 타임스탬프. 단일 세션이나 종료 시점이 정해지지 않은 이벤트의 경우 `null`일 수 있다.
    </ResponseField>

    <ResponseField name="items[].location" type="string">
      이벤트 장소나 위치에 대한 사람이 읽기 쉬운 설명. 해당하는 경우 온라인 형식을 나타낼 수 있다.
    </ResponseField>

    <ResponseField name="items[].organizer" type="object">
      이벤트를 주최하는 주체의 요약.

      <Expandable title="주최자 필드">
        <ResponseField name="items[].organizer.name" type="string">
          주최 기업 또는 개인의 표시 이름.
        </ResponseField>

        <ResponseField name="items[].organizer.handle" type="string">
          주최자의 로켓펀치 핸들. 전체 주최자 프로필 상세를 조회하려면 [Companies API](/api/reference/companies)와 함께 사용한다.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer" required>
  모든 페이지에 걸쳐 쿼리에 일치하는 이벤트 총 개수.
</ResponseField>

<ResponseField name="page" type="integer" required>
  이 응답에 반영된 현재 페이지 번호.
</ResponseField>

<ResponseField name="per_page" type="integer" required>
  이 응답에 반영된 페이지당 결과 수.
</ResponseField>

## 오류 응답

| HTTP 상태 | 의미 |
| --- | --- |
| `401 Unauthorized` | `X-RP-API-Key`가 없거나 유효하지 않다. |
| `429 Too Many Requests` | 플랜의 요청 한도를 초과했다. |
| `500 Internal Server Error` | 예상치 못한 서버 측 오류가 발생했다. 지수 백오프로 재시도한다. |

<Tip>
  Events 엔드포인트로 이벤트 애그리게이터를 만들거나 한국의 다가오는 IT 이벤트에 대한 알림을 사용자에게 시의적절하게 보낸다. 관련 `keyword`로 엔드포인트를 주기적으로 폴링하고 `start_date` 값을 현재 날짜와 비교해 UI에 다가오는 이벤트만 노출한다.
</Tip>
