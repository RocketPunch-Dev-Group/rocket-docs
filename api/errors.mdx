로켓펀치 API는 표준 HTTP 상태 코드를 사용하고 모든 실패에 대해 일관된 JSON 오류 본문을 반환한다. 덕분에 오류를 프로그래밍 방식으로 처리하기 쉽다. 자유 형식 오류 메시지를 파싱하지 않고도 `status` 필드나 `code` 문자열로 코드를 분기할 수 있다. 이 페이지는 연동에서 마주칠 수 있는 모든 오류와 해결 방법을 다룬다.

## 오류 응답 형식

모든 오류 응답은 오류 유형과 관계없이 동일한 JSON 구조를 공유한다. 기계가 읽을 수 있는 오류 식별자는 `code` 필드를, 사람이 읽을 수 있는 설명은 `message`를 확인한다.

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key",
    "status": 401
  }
}
```

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `code` | string | 기계가 읽을 수 있는 오류 식별자 (예: `UNAUTHORIZED`, `NOT_FOUND`) |
| `message` | string | 사람이 읽을 수 있는 오류 설명 |
| `status` | integer | HTTP 상태 코드. 응답 상태 라인에도 반영됨 |

## HTTP 상태 코드

아래 표는 API가 반환하는 모든 상태 코드, 의미, 그리고 취할 첫 조치를 나열한다.

| 상태 | 코드 문자열 | 의미 | 조치 |
| --- | --- | --- | --- |
| `400` | `BAD_REQUEST` | 요청에 잘못되거나 누락된 파라미터가 있다. | 엔드포인트 레퍼런스와 쿼리 파라미터를 대조한다. `message` 필드에서 어떤 파라미터가 잘못됐는지 확인한다. |
| `401` | `UNAUTHORIZED` | `X-RP-API-Key` 헤더가 없거나, 형식이 잘못됐거나, 폐기된 키를 가리킨다. | 헤더가 있는지, 키가 `rp_app_`로 시작하는지 확인한다. 필요하면 [developers.rocketpunch.com](https://developers.rocketpunch.com)에서 재발급한다. |
| `403` | `FORBIDDEN` | API 키는 유효하지만 이 리소스에 접근할 권한이 없다. | 앱에 필요한 스코프가 활성화되어 있는지 확인한다. OAuth를 사용한다면 사용자가 요청에 필요한 스코프를 부여했는지 확인한다. |
| `404` | `NOT_FOUND` | 요청한 리소스가 존재하지 않는다. | 엔드포인트 경로와 URL의 리소스 식별자를 확인한다. 리소스가 삭제됐거나 ID가 잘못됐을 수 있다. |
| `422` | `UNPROCESSABLE_ENTITY` | 요청 형식은 올바르지만 값이 검증에 실패했다. | 범위를 벗어난 `page` 값이나 지원되지 않는 `lang` 코드 같은 필드 수준 상세는 `message`를 읽는다. |
| `429` | `RATE_LIMIT_EXCEEDED` | 앱이 현재 창의 요청 할당량을 초과했다. | `Retry-After` 헤더에 표시된 시간만큼 기다린 뒤 재시도한다. 재시도 전략은 [레이트 리밋](/api/rate-limits) 페이지를 참고한다. |
| `500` | `INTERNAL_SERVER_ERROR` | 로켓펀치 서버에서 예상치 못한 오류가 발생했다. | 짧은 지연 후 지수 백오프로 재시도한다. 오류가 지속되면 [support@rocketpunch.com](mailto:support@rocketpunch.com)으로 문의한다. |
| `503` | `SERVICE_UNAVAILABLE` | 보통 점검 중에 API를 일시적으로 이용할 수 없다. | 잠시 후 재시도한다. 진행 중인 인시던트가 있는지 개발자 포털을 확인한다. |

## 흔한 문제와 해결책

### 유효하지 않거나 누락된 API 키 (401)

`401` 오류의 가장 흔한 원인은 누락되거나 형식이 잘못된 `X-RP-API-Key` 헤더다.

**헤더 이름을 확인한다.** 헤더는 정확히 `X-RP-API-Key`여야 한다(HTTP에서 대소문자 구분 없음이지만, 일부 HTTP 클라이언트의 버그를 피하려면 정확히 일치시킨다).

**키 형식을 확인한다.** App Key는 `rp_app_xxxxxxxxxxxxxxxx` 형식을 따른다. 키가 `rp_app_`로 시작하지 않는다면 개발자 포털에서 잘못된 값을 복사했을 수 있다.

**유출된 키를 재발급한다.** 키가 노출된 것이 의심되면 [developers.rocketpunch.com](https://developers.rocketpunch.com)에서 앱 설정을 열어 키를 교체한다. 새 값으로 환경 변수나 시크릿 매니저를 업데이트한다.

```bash
# 올바름 — 유효한 형식의 헤더 존재
curl "https://openapi.rocketpunch.com/api/v1/jobs" \
  -H "X-RP-API-Key: rp_app_xxxxxxxxxxxxxxxx"

# 잘못됨 — 헤더 이름 오타
curl "https://openapi.rocketpunch.com/api/v1/jobs" \
  -H "X-API-Key: rp_app_xxxxxxxxxxxxxxxx"
```

### 지원되지 않는 언어 코드 (400 / 422)

`/api/v1/jobs`, `/api/v1/companies` 같은 엔드포인트의 `lang` 파라미터는 특정 로케일 코드를 받는다. 유효하지 않은 값을 전달하면 `400` 또는 `422` 오류를 반환한다.

API는 11개 언어를 지원한다. IETF 언어 태그 형식을 사용한다:

| 언어 | `lang` 값 |
| --- | --- |
| 한국어 | `ko` |
| 영어 | `en` |
| 일본어 | `ja` |
| 중국어(간체) | `zh-hans` |
| 중국어(번체) | `zh-hant` |
| 스페인어 | `es` |
| 프랑스어 | `fr` |
| 독일어 | `de` |
| 포르투갈어 | `pt` |
| 인도네시아어 | `id` |
| 베트남어 | `vi` |

### 범위를 벗어난 페이지네이션 (400 / 422)

허용 범위를 벗어난 `page` 또는 `per_page` 값을 요청하면 API가 오류를 반환한다. `per_page`는 엔드포인트별 문서화된 최댓값(보통 100) 이내로 유지하고, 결과의 마지막 페이지를 넘는 페이지 번호를 요청하지 않는다.

```bash
# 응답 엔벨로프에서 total pages를 확인하고
# page >= total_pages일 때 페이지네이션을 중단한다
```

### OAuth 토큰 오류 (401 / 403)

Bearer 토큰 사용 시 `401`을 받으면 토큰이 만료됐을 수 있다. `refresh_token`을 사용해 새 액세스 토큰을 획득한다. 갱신 흐름은 [Auth Flow 가이드](/api/oauth/flow)를 참고한다.

Bearer 토큰에서의 `403`은 보통 호출하는 엔드포인트에 필요한 스코프 없이 토큰이 발급된 경우다. 사용자를 인증 흐름으로 다시 안내하고 누락된 스코프를 요청한다.

## 코드에서 오류 처리하기

프로그래밍 방식 분기를 위해 `error.code` 필드를 읽고, `error.message`를 로그나 모니터링 시스템에 노출하도록 오류 처리를 구성한다.

<CodeGroup>

```javascript Node.js
async function callAPI(endpoint, params) {
  const url = new URL(`https://openapi.rocketpunch.com${endpoint}`);
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));

  const response = await fetch(url, {
    headers: { "X-RP-API-Key": process.env.RP_API_KEY },
  });

  if (!response.ok) {
    const { error } = await response.json();

    switch (error.code) {
      case "UNAUTHORIZED":
        throw new Error("Check your RP_API_KEY environment variable.");
      case "RATE_LIMIT_EXCEEDED":
        // 호출자가 재시도 로직을 구현해야 함
        throw new RateLimitError(error.message);
      case "NOT_FOUND":
        return null; // 빈 결과로 처리
      default:
        throw new Error(`API error ${error.status}: ${error.message}`);
    }
  }

  return response.json();
}
```

```python Python
import os
import requests

def call_api(endpoint, params):
    url = f"https://openapi.rocketpunch.com{endpoint}"
    response = requests.get(
        url,
        params=params,
        headers={"X-RP-API-Key": os.environ["RP_API_KEY"]},
    )

    if not response.ok:
        error = response.json().get("error", {})
        code = error.get("code")
        message = error.get("message", "Unknown error")

        if code == "UNAUTHORIZED":
            raise ValueError("Check your RP_API_KEY environment variable.")
        elif code == "RATE_LIMIT_EXCEEDED":
            raise RateLimitError(message)  # 호출자가 재시도 처리
        elif code == "NOT_FOUND":
            return None  # 빈 결과로 처리
        else:
            raise Exception(f"API error {response.status_code}: {message}")

    return response.json()
```

</CodeGroup>

## 도움받기

여기서 다루지 않은 오류를 만나거나 재시도 후에도 `500`이 지속되면 로켓펀치 개발자 지원팀에 연락한다.

- **이메일:** [support@rocketpunch.com](mailto:support@rocketpunch.com)
- **개발자 포털:** [developers.rocketpunch.com](https://developers.rocketpunch.com)

지원팀에 문의할 때는 호출한 엔드포인트, 전체 오류 응답 본문, 요청 타임스탬프를 포함한다. 이렇게 하면 팀이 서버 측 문제를 빠르게 진단하는 데 도움이 된다.
