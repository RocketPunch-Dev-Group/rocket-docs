로켓펀치 OAuth 2.0 흐름 구현은 네 단계로 이뤄진다. 사용자를 로켓펀치로 리디렉션하고, 콜백 URL에서 인증 코드를 받고, 그 코드를 액세스 토큰으로 교환하고, 토큰을 사용해 사용자 범위의 API를 호출한다. 이 가이드는 각 단계를 동작하는 코드 예시와 함께 설명한다.

시작하기 전에 [OAuth 설정](/api/oauth/overview)을 완료했는지 확인한다. **Client ID**, **Client Secret**, 등록된 **리디렉션 URI**가 필요하다.

<Warning>
  Client Secret을 클라이언트 사이드 JavaScript, 모바일 앱, 공개 소스 코드에 절대 노출하지 않는다. 모든 토큰 교환 요청은 서버에서 시작되어야 한다.
</Warning>

## 1단계: 사용자를 인증 URL로 리디렉션

사용자가 로켓펀치 계정을 앱에 연결하려 할 때, 로켓펀치의 인증 엔드포인트로 리디렉션한다. 필수 쿼리 파라미터로 URL을 구성해 사용자를 그곳으로 보낸다.

**인증 엔드포인트:**

```text
https://www.rocketpunch.com/oauth/authorize
```

**필수 파라미터:**

| 파라미터 | 값 | 설명 |
| --- | --- | --- |
| `client_id` | 앱의 Client ID | 애플리케이션을 식별 |
| `redirect_uri` | 콜백 URL | 앱 설정에 등록된 URI와 일치해야 함 |
| `response_type` | `code` | 인증 코드를 요청 |
| `scope` | 공백으로 구분된 스코프 목록 | 앱이 요청하는 권한 |

**이용 가능한 스코프:**

| 스코프 | 설명 |
| --- | --- |
| `profile:read` | 사용자의 공개 프로필 정보 읽기 |
| `profile:email` | 사용자의 이메일 주소 접근 |

<CodeGroup>

```javascript Node.js
const params = new URLSearchParams({
  client_id: process.env.RP_CLIENT_ID,
  redirect_uri: "https://yourapp.com/auth/rocketpunch/callback",
  response_type: "code",
  scope: "profile:read profile:email",
});

const authorizationUrl = `https://www.rocketpunch.com/oauth/authorize?${params.toString()}`;

// Express 라우트 핸들러에서:
res.redirect(authorizationUrl);
```

```python Python
import os
from urllib.parse import urlencode

params = urlencode({
    "client_id": os.environ["RP_CLIENT_ID"],
    "redirect_uri": "https://yourapp.com/auth/rocketpunch/callback",
    "response_type": "code",
    "scope": "profile:read profile:email",
})

authorization_url = f"https://www.rocketpunch.com/oauth/authorize?{params}"

# Flask 라우트 핸들러에서:
return redirect(authorization_url)
```

```bash cURL (테스트용)
curl -G "https://www.rocketpunch.com/oauth/authorize" \
  --data-urlencode "client_id=YOUR_CLIENT_ID" \
  --data-urlencode "redirect_uri=https://yourapp.com/auth/rocketpunch/callback" \
  --data-urlencode "response_type=code" \
  --data-urlencode "scope=profile:read profile:email"
```

</CodeGroup>

## 2단계: 사용자가 동의를 부여

리디렉션 후에는 로켓펀치가 이어받는다. 사용자는 (로그인되어 있지 않다면) 로그인 화면을 본 뒤, 앱 이름과 요청한 권한을 보여주는 동의 화면을 본다.

- 사용자가 **승인**하면 로켓펀치가 `?code=` 쿼리 파라미터와 함께 사용자를 `redirect_uri`로 리디렉션한다.
- 사용자가 접근을 **거부**하거나 취소하면 로켓펀치가 `?error=access_denied` 파라미터와 함께 `redirect_uri`로 리디렉션한다.

콜백 핸들러는 두 결과를 모두 확인해야 한다.

## 3단계: 인증 코드 받기

로켓펀치는 동의 결정 후 사용자를 `redirect_uri`로 리디렉션한다. 쿼리 문자열에서 `code` 파라미터를 추출해 토큰 교환 로직으로 전달한다.

```text
https://yourapp.com/auth/rocketpunch/callback?code=AUTH_CODE_HERE
```

<CodeGroup>

```javascript Node.js (Express)
app.get("/auth/rocketpunch/callback", async (req, res) => {
  const { code, error } = req.query;

  if (error) {
    // 사용자가 접근을 거부했거나 오류가 발생함
    return res.redirect("/connect-failed");
  }

  if (!code) {
    return res.status(400).send("Missing authorization code");
  }

  // 토큰 교환으로 진행 (4단계)
  const token = await exchangeCodeForToken(code);
  // 토큰을 저장하고 사용자를 리디렉션
  req.session.accessToken = token.access_token;
  res.redirect("/dashboard");
});
```

```python Python (Flask)
from flask import request, redirect, session

@app.route("/auth/rocketpunch/callback")
def rocketpunch_callback():
    error = request.args.get("error")
    code = request.args.get("code")

    if error:
        return redirect("/connect-failed")

    if not code:
        return "Missing authorization code", 400

    # 토큰 교환으로 진행 (4단계)
    token = exchange_code_for_token(code)
    session["access_token"] = token["access_token"]
    return redirect("/dashboard")
```

</CodeGroup>

## 4단계: 코드를 액세스 토큰으로 교환

인증 코드는 단기이며 일회용이다. 서버가 로켓펀치 토큰 엔드포인트로 POST 요청을 보내 이를 액세스 토큰으로 교환해야 한다.

**토큰 엔드포인트:**

```text
POST https://www.rocketpunch.com/oauth/token
```

**필수 파라미터:**

| 파라미터 | 값 |
| --- | --- |
| `grant_type` | `authorization_code` |
| `code` | 3단계의 인증 코드 |
| `client_id` | 앱의 Client ID |
| `client_secret` | 앱의 Client Secret |
| `redirect_uri` | 1단계에서 사용한 동일한 리디렉션 URI |

<CodeGroup>

```javascript Node.js
async function exchangeCodeForToken(code) {
  const response = await fetch("https://www.rocketpunch.com/oauth/token", {
    method: "POST",
    headers: {
      "Content-Type": "application/x-www-form-urlencoded",
    },
    body: new URLSearchParams({
      grant_type: "authorization_code",
      code: code,
      client_id: process.env.RP_CLIENT_ID,
      client_secret: process.env.RP_CLIENT_SECRET,
      redirect_uri: "https://yourapp.com/auth/rocketpunch/callback",
    }),
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`Token exchange failed: ${error.error}`);
  }

  return response.json();
  // 반환: { access_token, token_type, expires_in, refresh_token, scope }
}
```

```python Python
import os
import requests

def exchange_code_for_token(code):
    response = requests.post(
        "https://www.rocketpunch.com/oauth/token",
        data={
            "grant_type": "authorization_code",
            "code": code,
            "client_id": os.environ["RP_CLIENT_ID"],
            "client_secret": os.environ["RP_CLIENT_SECRET"],
            "redirect_uri": "https://yourapp.com/auth/rocketpunch/callback",
        },
        headers={"Content-Type": "application/x-www-form-urlencoded"},
    )
    response.raise_for_status()
    return response.json()
    # 반환: { access_token, token_type, expires_in, refresh_token, scope }
```

```bash cURL
curl -X POST "https://www.rocketpunch.com/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code" \
  -d "code=AUTH_CODE_HERE" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "redirect_uri=https://yourapp.com/auth/rocketpunch/callback"
```

</CodeGroup>

성공한 응답은 액세스 토큰이 담긴 JSON 객체를 반환한다:

```json
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "def50200abc...",
  "scope": "profile:read profile:email"
}
```

`access_token`과 `refresh_token`을 백엔드에 안전하게 저장한다. 브라우저의 `localStorage`나 모바일 앱의 보호되지 않은 저장소에는 절대 저장하지 않는다.

## 5단계: 액세스 토큰 사용

모든 사용자별 API 요청의 `Authorization` 헤더에 액세스 토큰을 포함한다. `Bearer` 스킴을 사용한다.

<CodeGroup>

```javascript Node.js
async function getUserProfile(accessToken) {
  const response = await fetch(
    "https://openapi.rocketpunch.com/api/v1/profiles/me",
    {
      headers: {
        Authorization: `Bearer ${accessToken}`,
        "Content-Type": "application/json",
      },
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`API request failed: ${error.error.message}`);
  }

  return response.json();
}
```

```python Python
import requests

def get_user_profile(access_token):
    response = requests.get(
        "https://openapi.rocketpunch.com/api/v1/profiles/me",
        headers={
            "Authorization": f"Bearer {access_token}",
            "Content-Type": "application/json",
        },
    )
    response.raise_for_status()
    return response.json()
```

```bash cURL
curl "https://openapi.rocketpunch.com/api/v1/profiles/me" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

</CodeGroup>

## 토큰 갱신

액세스 토큰은 `expires_in`에 표시된 기간(초) 후 만료된다. 토큰이 만료되면 `refresh_token`을 사용해 사용자가 동의 흐름을 다시 거치지 않고도 새 액세스 토큰을 획득한다.

<CodeGroup>

```javascript Node.js
async function refreshAccessToken(refreshToken) {
  const response = await fetch("https://www.rocketpunch.com/oauth/token", {
    method: "POST",
    headers: {
      "Content-Type": "application/x-www-form-urlencoded",
    },
    body: new URLSearchParams({
      grant_type: "refresh_token",
      refresh_token: refreshToken,
      client_id: process.env.RP_CLIENT_ID,
      client_secret: process.env.RP_CLIENT_SECRET,
    }),
  });

  if (!response.ok) {
    // 리프레시 토큰이 만료되었을 수 있음. 사용자에게 재인증을 요청
    throw new Error("Token refresh failed. Re-authorization required.");
  }

  return response.json();
}
```

```python Python
def refresh_access_token(refresh_token):
    response = requests.post(
        "https://www.rocketpunch.com/oauth/token",
        data={
            "grant_type": "refresh_token",
            "refresh_token": refresh_token,
            "client_id": os.environ["RP_CLIENT_ID"],
            "client_secret": os.environ["RP_CLIENT_SECRET"],
        },
        headers={"Content-Type": "application/x-www-form-urlencoded"},
    )
    if not response.ok:
        # 리프레시 토큰이 만료되었을 수 있음. 사용자에게 재인증을 요청
        raise Exception("Token refresh failed. Re-authorization required.")
    return response.json()
```

</CodeGroup>

<Tip>
  `401 Unauthorized` 응답을 기다리기보다 만료 전에 액세스 토큰을 선제적으로 갱신한다. `expires_in` 값을 추적하고 몇 분 일찍 갱신을 예약해 사용자 세션을 매끄럽게 유지한다.
</Tip>

리프레시 토큰 자체가 만료되었거나 폐기되었다면, 사용자를 전체 인증 흐름으로 다시 안내해야 한다.
